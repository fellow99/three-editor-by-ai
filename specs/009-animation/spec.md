# 009-Animation 动画系统模块 - 规格文档

**模块版本**: 1.0.0  
**生成日期**: 2026-04-14  
**最后更新**: 2026-04-14

---

## 1. 模块概述

### 1.1 模块定位

动画系统模块负责 3D 模型中内嵌动画的自动检测、播放控制、状态管理和序列化持久化。它是编辑器处理动态模型的核心能力，使用户能够无缝使用带有动画的 3D 资源。

### 1.2 模块职责

- **动画自动检测**: 加载模型时自动识别并提取动画片段
- **动画播放控制**: 支持动画的选择、播放、暂停和切换
- **运行时状态管理**: 每个对象独立的动画混合器和动作管理
- **序列化集成**: 动画状态在场景保存和加载时的完整持久化
- **UI 交互支持**: 提供动画选择和播放控制的用户界面

### 1.3 核心价值

- 零配置动画发现，模型加载即可用
- 每对象独立动画控制，互不干扰
- 动画状态完整序列化，保存加载不丢失
- 运行时状态与可序列化状态清晰分离

## 2. 动画自动检测规格

### 2.1 检测时机

**触发条件**: 3D 模型资源加载完成时

**检测对象**: 所有包含动画数据的模型文件

**处理流程**:
1. 资源加载器完成模型加载
2. 检查加载结果中是否存在动画片段数组
3. 若存在动画片段，将其存储到模型根对象上
4. 为模型根对象创建独立的动画混合器
5. 标记该对象为"可动画对象"

### 2.2 动画数据存储

**存储位置**: 动画相关运行时数据直接存储在对象实例上

**存储内容**:
- 动画片段数组: 存储所有可用的动画片段
- 动画混合器: 用于驱动该对象动画的混合器实例
- 当前动作: 正在播放的动画动作引用

**设计原则**:
- 运行时数据不进入可序列化字段
- 避免污染用户自定义数据区域
- 直接对象引用提升访问效率

### 2.3 检测边界

**支持范围**:
- 包含动画片段的模型文件
- 模型根对象及其子对象层级

**不支持范围**:
- 无动画数据的静态模型
- 运行时动态生成的动画片段
- 外部动画文件引用

## 3. 动画播放控制规格

### 3.1 动画选择

**选择机制**: 通过索引值选择要播放的动画片段

**索引存储**: 选中的动画索引存储在对象的用户自定义数据中

**切换行为**:
1. 停止当前正在播放的动画动作
2. 根据新索引获取对应的动画片段
3. 创建新的动画动作并开始播放
4. 更新索引值到用户自定义数据

### 3.2 播放控制

**支持的操作**:
- 播放: 开始播放选中的动画
- 停止: 停止当前动画播放
- 切换: 切换到另一个动画片段

**播放状态**:
- 每个对象独立维护自己的播放状态
- 多个对象的动画可以同时播放
- 动画播放互不影响

### 3.3 动画驱动

**驱动机制**: 渲染循环中统一更新所有动画混合器

**更新频率**: 每帧执行一次

**更新流程**:
1. 遍历场景中的所有对象
2. 识别带有动画混合器的对象
3. 调用混合器的更新方法，传入时间增量
4. 动画状态自动推进到下一帧

**时间处理**:
- 使用帧间时间增量确保动画速度与帧率无关
- 高帧率和低帧率下动画播放速度一致

## 4. UI 组件规格

### 4.1 组件概述

**组件名称**: 动画属性面板

**显示条件**: 当选中对象包含动画片段时自动显示

**隐藏条件**: 当选中对象无动画或未选中任何对象时隐藏

### 4.2 功能元素

**动画列表**:
- 下拉选择器展示所有可用动画片段
- 按动画片段名称排序显示
- 当前选中的动画高亮标识

**播放控制**:
- 播放/暂停切换按钮
- 按钮状态反映当前播放状态
- 点击切换播放状态

**状态显示**:
- 当前播放的动画名称
- 动画总数量
- 播放状态指示

### 4.3 交互行为

**选择动画**:
1. 用户从下拉列表选择动画
2. 系统停止当前动画
3. 系统播放新选择的动画
4. 更新对象的动画索引值

**播放/暂停**:
1. 用户点击播放/暂停按钮
2. 系统切换当前动画的播放状态
3. 按钮图标和状态同步更新

## 5. 序列化集成规格

### 5.1 导出行为

**序列化内容**:
- 动画索引值: 记录当前选中的动画
- 用户自定义数据中的其他字段

**不序列化内容**:
- 动画混合器实例
- 当前动画动作引用
- 动画片段数组
- 所有运行时创建的对象引用

**序列化原则**:
- 仅序列化可重建状态的最小数据集
- 运行时对象不进入序列化数据
- 保持序列化数据简洁和可读

### 5.2 导入行为

**恢复流程**:
1. 从序列化数据中读取对象信息
2. 恢复对象的基本属性和用户自定义数据
3. 检查用户自定义数据中是否存在动画索引
4. 若对象包含动画片段且存在动画索引:
   - 创建新的动画混合器
   - 根据索引加载对应的动画片段
   - 创建动画动作并开始播放
5. 完成对象恢复

**恢复条件**:
- 对象必须包含动画片段
- 用户自定义数据中必须存在有效的动画索引
- 动画索引必须在动画片段数组范围内

### 5.3 状态一致性

**保存前状态**:
- 动画正在播放
- 动画索引为 2
- 混合器正在运行

**保存后数据**:
- 仅保存动画索引值 2
- 不保存混合器和动作状态

**加载后状态**:
- 动画索引恢复为 2
- 混合器重新创建
- 索引为 2 的动画自动开始播放

## 6. 动画混合器管理规格

### 6.1 混合器生命周期

**创建时机**: 模型加载完成且包含动画片段时

**销毁时机**: 对象从场景中移除时

**复用策略**: 同一对象的混合器在整个生命周期内复用

### 6.2 动作管理

**动作创建**: 通过混合器从动画片段创建动作

**动作播放**: 调用动作的播放方法开始动画

**动作停止**: 调用动作的停止方法结束动画

**动作切换**: 先停止旧动作，再创建并播放新动作

### 6.3 多动画支持

**单对象单动画**: 同一时刻一个对象只能播放一个动画

**多对象多动画**: 不同对象可以同时播放各自的动画

**动画混合**: 预留动画混合能力，当前版本不支持

## 7. API 接口规格

### 7.1 场景级动画驱动

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `updateAllMixers(delta)` | delta: number | void | 更新场景中所有动画混合器 |

### 7.2 对象级动画控制

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `playAnimation(object, index)` | object: Object3D, index: number | void | 播放指定对象的指定动画 |
| `stopAnimation(object)` | object: Object3D | void | 停止指定对象的动画播放 |
| `getAnimationClips(object)` | object: Object3D | Array | 获取对象的动画片段列表 |
| `hasAnimations(object)` | object: Object3D | boolean | 检查对象是否包含动画 |

### 7.3 序列化相关

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `exportAnimationState(object)` | object: Object3D | Object | 导出动画状态 |
| `importAnimationState(object, state)` | object: Object3D, state: Object | void | 导入动画状态 |

## 8. 事件系统规格

### 8.1 触发的事件

| 事件名 | 触发时机 | 数据 |
|--------|----------|------|
| `animation-selected` | 用户选择新动画 | `{ object, clipIndex, clipName }` |
| `animation-played` | 动画开始播放 | `{ object, clipIndex }` |
| `animation-stopped` | 动画停止播放 | `{ object, clipIndex }` |
| `animation-clips-detected` | 检测到模型动画 | `{ object, clipCount, clipNames }` |

### 8.2 监听的事件

| 事件名 | 来源 | 响应动作 |
|--------|------|----------|
| `object-added` | 对象管理器 | 检查新对象是否包含动画 |
| `object-removed` | 对象管理器 | 清理对象的动画混合器 |
| `scene-loaded` | 场景查看器 | 恢复所有对象的动画状态 |

## 9. 性能优化规格

### 9.1 更新优化

**遍历优化**:
- 仅遍历场景中有动画混合器的对象
- 无动画对象跳过更新调用
- 使用场景遍历方法自动过滤

**时间计算**:
- 使用高精度时间戳计算帧间增量
- 避免时间跳跃导致的动画异常
- 支持暂停和恢复时的时间连续性

### 9.2 内存优化

**混合器复用**:
- 同一对象不重复创建混合器
- 对象移除时及时释放混合器资源

**动作管理**:
- 切换动画时释放旧动作引用
- 避免动作对象累积导致内存泄漏

### 9.3 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单对象动画更新耗时 | < 0.1ms | 单个混合器更新时间 |
| 10 对象动画更新耗时 | < 1ms | 10 个混合器总更新时间 |
| 动画切换响应时间 | < 50ms | 从选择到播放的延迟 |
| 内存增长（每对象） | < 100KB | 每个动画对象的额外内存 |

## 10. 错误处理规格

### 10.1 动画不存在

**场景**: 尝试播放不存在的动画索引

**处理**:
- 记录警告日志
- 不执行播放操作
- 保持当前动画状态不变

### 10.2 对象无动画

**场景**: 对无动画对象执行动画操作

**处理**:
- 静默忽略操作
- 不抛出异常
- UI 面板不显示

### 10.3 混合器创建失败

**场景**: 动画混合器创建过程中发生错误

**处理**:
- 记录错误日志
- 触发错误事件
- 不影响其他对象的动画播放

### 10.4 序列化数据异常

**场景**: 加载的序列化数据中动画索引无效

**处理**:
- 忽略无效的动画索引
- 不自动播放任何动画
- 用户可手动选择动画

## 11. 依赖关系

### 11.1 内部依赖

```
动画系统 → 资源加载器（获取动画片段）
动画系统 → 场景查看器（渲染循环驱动）
动画系统 → 对象管理器（对象生命周期）
动画系统 → 属性面板（UI 展示）
```

### 11.2 外部依赖

| 依赖 | 用途 |
|------|------|
| 3D 引擎核心库 | AnimationMixer、AnimationClip、AnimationAction |
| 3D 引擎加载器 | GLTF 模型加载和动画提取 |

## 12. 使用示例

### 12.1 加载带动画的模型

```
1. 用户拖拽模型文件到场景
2. 资源加载器加载模型
3. 系统自动检测动画片段
4. 动画片段存储到对象上
5. 动画混合器自动创建
```

### 12.2 播放动画

```
1. 用户选中带动画的对象
2. 动画属性面板自动显示
3. 用户从下拉列表选择动画
4. 系统播放选中的动画
5. 动画索引保存到对象数据
```

### 12.3 保存和加载场景

```
1. 用户保存场景
2. 动画索引被序列化到文件
3. 运行时对象不被序列化
4. 用户加载场景
5. 动画状态自动恢复
6. 选中的动画自动播放
```
