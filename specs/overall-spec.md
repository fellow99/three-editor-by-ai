# Three Editor by AI - 整体规格文档

**生成日期**: 2026-04-14  
**项目版本**: 0.2.0  
**文档类型**: 产品规格说明

---

## 1. 产品概述

### 1.1 产品定位

Three Editor by AI 是一个基于 Web 的现代化 3D 场景编辑器，允许用户在浏览器中创建、编辑和管理 3D 场景，无需安装任何桌面软件。

### 1.2 目标用户

- **3D 内容创作者**: 快速原型设计和场景预览
- **Web 开发者**: 在 Web 应用中集成 3D 内容
- **教育工作者**: 3D 可视化和教学演示
- **建筑师/设计师**: 简单的 3D 场景布局和展示

### 1.3 核心价值

- **零安装**: 基于浏览器，打开即用
- **直观交互**: 拖拽式操作，降低 3D 编辑门槛
- **实时预览**: 所见即所得的编辑体验
- **灵活扩展**: 支持多种 3D 格式、后处理效果和虚拟文件系统

## 2. 功能规格

### 2.1 场景管理

#### 2.1.1 场景创建
- 用户可创建新的空场景
- 新场景包含默认相机位置和基础光照设置
- 支持设置场景背景色

#### 2.1.2 场景加载
- 支持从虚拟文件系统加载已保存的场景
- 支持从本地文件导入场景（JSON 格式）
- 加载时完整恢复场景状态，包括所有对象、材质、变换、用户数据和动画

#### 2.1.3 场景保存
- 支持保存到虚拟文件系统
- 支持导出为 JSON 文件下载到本地
- 保存内容包括：所有对象及其材质、变换属性、用户自定义数据、动画索引

#### 2.1.4 场景导出
- 导出为 JSON 格式
- 导出时保留完整的场景层级结构和所有可序列化数据

**验收标准**:
- [ ] 新建空场景后，视图中显示默认相机和光照
- [ ] 保存场景后，再次加载可完整恢复所有对象和属性
- [ ] 导出 JSON 文件可在本地存储并重新导入
- [ ] 加载非本编辑器生成的合法 JSON 场景文件时，给出明确错误提示

### 2.2 对象管理

#### 2.2.1 基础几何体创建
用户可通过拖拽方式创建以下几何体（共 13 种以上）：
- **立方体**: 可设置长、宽、高
- **球体**: 可设置半径、分段数
- **圆柱体**: 可设置顶底半径、高度、分段数
- **圆锥体**: 可设置半径、高度、分段数
- **平面**: 可设置宽度、高度、分段数
- **圆形**: 可设置半径、分段数
- **圆环**: 可设置主体半径、管道半径、分段数
- **管状体**: 可设置路径、半径、分段数
- **十二面体**: 可设置半径、细节级别
- **二十面体**: 可设置半径、细节级别
- **八面体**: 可设置半径、细节级别
- **四面体**: 可设置半径、细节级别

#### 2.2.2 灯光创建
用户可通过拖拽方式创建以下灯光（共 6 种）：
- **环境光**: 全局均匀照明
- **平行光**: 模拟远距离定向光源
- **点光源**: 向所有方向发射光线
- **聚光灯**: 锥形定向光源
- **半球光**: 天空与地面颜色渐变照明
- **矩形面光源**: 矩形区域发光体

#### 2.2.3 其他对象类型
- **相机**: 可添加到场景中作为视点参考
- **编组**: 将多个对象组合为层级结构
- **文本**: 3D 文字对象
- **精灵**: 始终面向相机的 2D 广告牌对象

#### 2.2.4 对象添加方式
- 所有对象仅支持通过拖拽方式添加到场景
- 拖拽来源包括：基础几何体面板、资源浏览器、虚拟文件浏览器、外部文件拖入
- 不提供点击按钮直接添加对象的方式

#### 2.2.5 对象选择
- 单击视图中对象可选中
- 通过场景层级树可选择对象
- 选中对象在视图中高亮显示

#### 2.2.6 对象锁定
- 可锁定对象以防止被选中或变换
- 锁定对象不参与射线检测
- 锁定状态随场景序列化保存和恢复

#### 2.2.7 对象标识
- 每个对象拥有全局唯一的 UUID 标识
- UUID 在对象创建时生成，序列化时保留

**验收标准**:
- [ ] 所有 13+ 种几何体可通过拖拽成功创建到场景中
- [ ] 所有 6 种灯光可通过拖拽创建并正确影响场景光照
- [ ] 拖拽添加的对象放置位置合理（当前视点附近）
- [ ] 锁定对象无法被射线选中或变换操作影响
- [ ] 对象 UUID 在保存和加载后保持一致

### 2.3 对象变换

#### 2.3.1 交互式变换
- **移动**: 通过变换 gizmo 沿 X/Y/Z 轴移动对象
- **旋转**: 通过变换 gizmo 绕 X/Y/Z 轴旋转对象
- **缩放**: 通过变换 gizmo 沿 X/Y/Z 轴缩放对象

#### 2.3.2 吸附功能
- **网格吸附**: 移动时按指定网格间距对齐
- **旋转吸附**: 旋转时按指定角度增量对齐
- **缩放吸附**: 缩放时按指定比例增量对齐

#### 2.3.3 轴约束
- 支持锁定单一轴（X/Y/Z），仅允许沿该轴变换
- 支持 Y 轴锁定模式，仅允许在 XZ 平面内操作

#### 2.3.4 坐标空间
- 支持世界坐标系变换
- 支持局部坐标系变换
- 可在两种坐标空间之间切换

#### 2.3.5 多选批量变换
- 支持同时选中多个对象
- 对多选对象执行统一的变换操作
- 变换 gizmo 作用于选中对象的包围盒中心

**验收标准**:
- [ ] 变换 gizmo 可正确显示并响应用户拖拽操作
- [ ] 网格吸附开启后，对象移动位置符合网格间距
- [ ] 轴约束开启后，对象仅沿指定轴变换
- [ ] 世界/局部坐标空间切换后，gizmo 方向正确更新
- [ ] 多选对象变换时，所有对象同步移动

---

### 2.4 撤销/重做

#### 2.4.1 历史记录
- 支持最多 50 步撤销操作
- 撤销和重做共享同一历史栈

#### 2.4.2 状态捕获
- 在变换 gizmo 拖拽开始时捕获对象当前状态
- 捕获内容包括：位置、旋转、缩放

#### 2.4.3 状态恢复
- 撤销操作将对象状态恢复到上一步
- 重做操作将对象状态恢复到撤销前的状态

#### 2.4.4 栈式管理
- 撤销操作从历史栈弹出状态到重做栈
- 重做操作从重做栈弹出状态回历史栈
- 执行新操作后，重做栈清空

**验收标准**:
- [ ] 连续执行 50 次变换后，可逐次撤销到初始状态
- [ ] 撤销后可重做恢复到撤销前状态
- [ ] 撤销后执行新操作，原重做历史被清空
- [ ] 撤销/重做操作响应时间小于 100ms

### 2.5 材质系统

#### 2.5.1 材质类型
支持 7 种以上材质类型，包括：
- 基础材质（不受光照影响）
- 标准物理材质（PBR）
- Phong 高光材质
- Lambert 漫反射材质
- 物理渲染材质（高级 PBR）
- 其他扩展材质类型

#### 2.5.2 标准属性
- **颜色**: 基础色、发射色等
- **粗糙度**: 表面粗糙程度（0-1）
- **金属度**: 金属质感程度（0-1）
- **透明度**: 材质透明程度
- **不透明度**: 材质不透明程度

#### 2.5.3 纹理贴图
支持以下纹理贴图类型：
- 漫反射贴图
- 法线贴图
- 粗糙度贴图
- 金属度贴图
- 发射贴图
- 环境光遮蔽贴图
- 高度贴图

#### 2.5.4 材质预览
- 材质属性变更时实时预览渲染效果
- 纹理贴图提供缩略图预览

#### 2.5.5 专家模式
- 提供专家模式，展示 16 种以上高级子属性
- 包括清漆层、涂层、透射、厚度等高级参数

#### 2.5.6 材质选择
- 支持从虚拟文件系统中选择并应用材质
- 支持材质保存和复用

**验收标准**:
- [ ] 切换材质类型后，对象外观正确更新
- [ ] 修改颜色、粗糙度、金属度等参数后，渲染效果实时变化
- [ ] 应用纹理贴图后，对象表面正确显示纹理
- [ ] 专家模式下可编辑 16+ 种高级属性
- [ ] 从文件系统选择材质可正确应用到选中对象

### 2.6 相机系统

#### 2.6.1 控制器模式
- **轨道模式**: 环绕目标点旋转观察
- **地图模式**: 平移和缩放，适合俯视场景
- **飞行模式**: 第一人称自由飞行

#### 2.6.2 控制器切换
- 支持在三种控制器模式之间热切换
- 切换时保持当前相机位置不变

#### 2.6.3 预设视角
- 提供多个预设视角位置（顶视、底视、前视、后视、左视、右视、等轴测等）
- 点击预设视角可快速跳转到对应位置

#### 2.6.4 相机过渡动画
- 支持相机飞行动画（从一个位置平滑过渡到另一个位置）
- 过渡过程可配置速度和曲线

#### 2.6.5 飞行控制
- 键盘控制：WASD 前后左右移动
- 键盘控制：QE 键上下移动
- 键盘控制：空格键/R 键向上飞行
- 键盘控制：方向键旋转视角
- 鼠标控制视角旋转
- 飞行速度可调节

#### 2.6.6 相机状态管理
- 可捕获当前相机状态（位置、朝向、FOV 等）
- 可恢复已保存的相机状态
- 支持命名导航检查点，保存和快速跳转到特定视角

**验收标准**:
- [ ] 三种控制器模式可正常切换，切换后操作行为符合预期
- [ ] 预设视角点击后相机平滑移动到目标位置
- [ ] 飞行模式下 WASD/QE/方向键控制正确响应
- [ ] 相机飞行动画平滑，无跳跃
- [ ] 命名导航检查点可保存和恢复

### 2.7 资产管理

#### 2.7.1 模型加载
- 支持 GLTF/GLB 格式（含压缩支持）
- 支持 OBJ 格式
- 支持 FBX 格式

#### 2.7.2 纹理加载
- 支持常见图片格式作为纹理（JPG、PNG 等）
- 支持压缩纹理格式

#### 2.7.3 资源缓存
- 基于文件名和大小的缓存机制
- 避免重复加载相同资源
- 缓存命中时直接返回已有资源实例

#### 2.7.4 预览缩略图
- 模型加载后自动生成预览缩略图
- 缩略图通过离屏渲染生成
- 缩略图在资源浏览器中显示

#### 2.7.5 批量加载
- 支持队列式批量资源加载
- 显示加载进度

#### 2.7.6 拖拽加载
- 支持从浏览器外部拖拽文件到场景
- 支持从文件浏览器拖拽文件到场景
- 拖拽后对象自动放置到当前视点位置

#### 2.7.7 加载方式
- 支持本地文件选择加载
- 支持 URL 地址加载

**验收标准**:
- [ ] GLTF/OBJ/FBX 模型可成功加载到场景中
- [ ] 相同文件第二次加载时直接使用缓存，不重复请求
- [ ] 模型加载后在资源浏览器中显示缩略图
- [ ] 从浏览器外部拖拽模型文件到场景可正确加载
- [ ] 批量加载多个资源时，进度显示正确

---

### 2.8 虚拟文件系统

#### 2.8.1 双模式支持
- **读写服务器模式**: 连接后端服务，支持文件读写操作
- **只读静态模式**: 使用静态文件资源，仅支持读取

#### 2.8.2 多驱动器配置
- 支持配置多个文件驱动器
- 每个驱动器可独立设置模式和路径

#### 2.8.3 文件浏览
- 提供文件浏览器组件
- 支持目录导航和层级展开
- 显示文件类型图标和基本信息

#### 2.8.4 文件操作
- 支持从文件浏览器拖拽文件到场景
- 支持文件保存对话框
- 支持文件加载对话框

#### 2.8.5 材质选择器
- 提供从虚拟文件系统选择材质的对话框
- 支持按目录浏览和筛选材质文件

**验收标准**:
- [ ] 文件浏览器可正确显示目录结构和文件列表
- [ ] 读写模式下可保存和加载文件
- [ ] 只读模式下仅可浏览和加载文件
- [ ] 从文件浏览器拖拽文件到场景可正确加载
- [ ] 多驱动器配置下，各驱动器内容独立显示

---

### 2.9 序列化

#### 2.9.1 导出格式
- 场景导出为 JSON 格式
- JSON 结构包含版本号和元数据

#### 2.9.2 导入格式
- 从 JSON 文件导入场景
- 导入时验证 JSON 格式和版本兼容性

#### 2.9.3 序列化内容
序列化时保留以下数据：
- 所有场景对象及其层级关系
- 对象材质及其属性
- 对象变换（位置、旋转、缩放）
- 用户自定义数据（userData）
- 动画索引

#### 2.9.4 排除内容
- 运行时临时数据不序列化
- 运行时生成的辅助对象不序列化
- 渲染器内部状态不序列化

#### 2.9.5 状态恢复
- 导入 JSON 后完整恢复场景状态
- 恢复后的场景与导出前一致

**验收标准**:
- [ ] 导出 JSON 文件包含所有对象、材质、变换、用户数据和动画索引
- [ ] 导出的 JSON 文件可重新导入并完整恢复场景
- [ ] 运行时临时数据不出现在导出的 JSON 中
- [ ] 导入非法 JSON 文件时给出明确错误提示

### 2.10 后处理

#### 2.10.1 效果链管理
- 支持多个后处理效果串联
- 支持动态添加和移除效果通道
- 提供统一的效果管理 API

#### 2.10.2 发光效果
- 支持 Bloom 发光效果
- 可调节发光强度、阈值、半径等参数

#### 2.10.3 抗锯齿
- 支持 FXAA 快速近似抗锯齿
- 可启用和禁用

#### 2.10.4 效果管理
- 效果启用/禁用切换
- 效果参数实时调节
- 效果按顺序执行

**验收标准**:
- [ ] Bloom 效果启用后，高亮区域产生发光效果
- [ ] FXAA 启用后，场景边缘锯齿明显减少
- [ ] 动态添加/移除效果通道后，渲染结果正确更新
- [ ] 多个效果同时启用时，按正确顺序执行

---

### 2.11 动画

#### 2.11.1 动画检测
- 模型加载时自动检测附带动画
- 动画列表在属性面板中显示

#### 2.11.2 动画选择
- 属性面板提供动画选择下拉菜单
- 可选择播放特定动画片段

#### 2.11.3 动画状态存储
- 当前播放的动画索引存储到对象用户数据中
- 动画索引随场景序列化保存

#### 2.11.4 动画驱动
- 所有活动动画由中央更新循环统一驱动
- 动画播放与场景渲染同步

#### 2.11.5 动画序列化
- 动画状态完全可序列化
- 场景加载后自动恢复动画状态

**验收标准**:
- [ ] 加载带动画的模型后，动画列表正确显示
- [ ] 选择不同动画片段可正确切换播放
- [ ] 保存场景后重新加载，动画状态正确恢复
- [ ] 多个带动画模型同时存在时，所有动画正常播放

## 3. 界面布局

### 3.1 整体布局

采用响应式三面板布局：
- **左侧面板**: 资源面板
- **中央区域**: 3D 视口
- **右侧面板**: 属性面板
- **顶部**: 工具栏
- **底部**: 状态栏

### 3.2 工具栏

Ribbon 风格菜单，包含：
- **文件菜单**: 新建、打开、保存、导出
- **编辑菜单**: 撤销、重做、删除
- **变换菜单**: 移动、旋转、缩放工具切换
- **视图菜单**: 控制器切换、预设视角
- **配置菜单**: 编辑器设置

### 3.3 左侧面板

资源面板，包含 4 个标签页：
- **场景树**: 场景对象层级树
- **资源浏览器**: 已加载资源浏览
- **基础对象浏览器**: 几何体和灯光浏览
- **文件浏览器**: 虚拟文件系统浏览

### 3.4 中央区域

3D 视口，包含：
- 3D 渲染区域
- 视口叠加控件（视口 gizmo、操作提示、性能统计、相机控制）

### 3.5 右侧面板

- **属性面板**: 单对象属性编辑（上下文敏感）
- **多选面板**: 多对象批量操作

### 3.6 底部状态栏

- 选中对象信息
- 相机位置信息
- 场景统计信息

## 4. 非功能规格

### 4.1 性能要求

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 帧率 (简单场景) | 60 FPS | 空场景 + 少量对象 |
| 帧率 (复杂场景) | 30+ FPS | 大量对象和材质 |
| 内存使用 | 稳定，无泄漏 | 长时间操作后对比 |
| 交互响应延迟 | < 100ms | 操作到视觉反馈 |

### 4.2 兼容性要求

#### 浏览器支持
- Chrome（最新版本）
- Firefox（最新版本）
- Edge（最新版本）
- Safari（最新版本）

#### WebGL 支持
- WebGL 2.0（必需）

### 4.3 布局要求

- 响应式三面板布局：左侧资源面板 / 中央视口 / 右侧属性面板
- 顶部工具栏
- 底部状态栏
- 单页面应用，无页面跳转

### 4.4 可用性要求

- 操作有明确的视觉反馈
- 错误提示清晰易懂
- 支持长时间使用（内存稳定）

### 4.5 可扩展性要求

- 支持添加新的几何体类型
- 支持添加新的材质类型
- 支持添加新的后处理效果
- 支持添加新的模型格式

## 8. 假设与约束

### 8.1 假设

- 用户具备基本的 3D 概念知识（坐标、变换、材质）
- 用户使用现代浏览器

- 不支持多人协作编辑（当前版本）

## 11. 术语表

| 术语 | 定义 |
|------|------|
| **场景** | 包含所有 3D 对象、灯光、相机的完整工作环境 |
| **对象** | 场景中的实体，包括几何体、灯光、相机等 |
| **变换** | 对象的位置、旋转、缩放属性 |
| **材质** | 定义对象外观的属性集合（颜色、纹理、粗糙度等） |
| **userData** | 对象的自定义数据字段，用于存储扩展属性 |
| **VFS** | Virtual File System，虚拟文件系统 |
| **GLTF** | GL Transmission Format，3D 模型交换格式 |
| **UUID** | Universally Unique Identifier，全局唯一标识符 |
| **Gizmo** | 交互式变换控件（移动/旋转/缩放的可视化手柄） |
| **PBR** | Physically Based Rendering，基于物理的渲染 |
| **Bloom** | 后处理发光效果 |
| **FXAA** | Fast Approximate Anti-Aliasing，快速近似抗锯齿 |
